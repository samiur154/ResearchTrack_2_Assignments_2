scripts.go_to_point.planning(goal)[source]

Set the appropriate behaviour depending on the current robot state, in orderd to reach the goal. The state machine keeps running until the goal is reached or the action is preempted (the goal gets cancelled).

Parameters

goal (PoseActionGoal) – (x,y,theta) goal pose

scripts.user_interface.main()[source]

This function initializes the ROS node and waits for the user to insert start or stop to control the robot, by relying on the rospy module.

The user message is passed to the service``user_interface``, advertised by go_to_point.
